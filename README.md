# HH.ru parsing pipeline

Пайплайн обработки сырых данных HH.ru с резюме.  
Построен на паттерне «цепочка ответственности» и запускается командой:

```bash
python source/app.py path/to/hh.csv
```

Можно указать имя таргета (колонка, которая станет `y` после обработки), по умолчанию: `salary`:

```bash
python source/app.py path/to/hh.csv --target salary
```

## Выходные файлы

1) **Рядом с исходным `hh.csv`** создаются:

- `x_data.npy` — матрица признаков (`float64`);
- `y_data.npy` — целевой вектор (значения из `--target`).

2) Дополнительно пайплайн сохраняет промежуточные датасеты (путь фиксированный, относительно текущей директории
   запуска):

- `data/processed/hh_prepared_raw.csv` — после парсинга/нормализации, до финального кодирования;
- `data/processed/hh_prepared_encoded.csv` — после финального кодирования, перед split на `X/y`.

## Что делает пайплайн

1. Загружает CSV, чистит текстовые артефакты и дубликаты.
2. Парсит из сырых колонок:
    - `Пол, возраст` -> `sex`, `age`;
    - `ЗП` -> `salary`, `currency`;
    - опыт работы -> `experience_years`;
    - `Образование и ВУЗ` -> `education_last_year`, `education_level`;
    - `Авто` -> `has_car`;
    - `Город` -> `city`, `relocation`, `business_trips`;
    - `График` -> нормализованный график работы;
    - должности -> `position`, `last_position`.

3. Нормализует/подготавливает отдельные поля (например, приводит зарплату к RUB по фиксированным курсам, нормализует
   текстовые поля позиций и т.п.).

4. Кодирует признаки и подчищает датасет:
    - бинарные фичи: `sex`, `relocation`, `has_car` -> 0/1;
    - должности: `position`, `last_position` -> encoding;
    - график: -> multi‑hot;
    - валюта и командировки: -> one‑hot;
    - образование: -> one‑hot;
    - удаляет строки с `business_trips == "unknown"` и `education_level == "unknown"`;
    - удаляет сырые текстовые поля и приводит признаки к числовому виду для дальнейшего анализа.

5. Делит датасет на `X` и `y` по колонке `--target` и сохраняет в `*.npy`.

### Warning

Небольшая часть строк удаляется в процессе очистки:

- дубликаты;
- записи с неопределенным уровнем образования;
- записи без информации о командировках.
